## ЧТО ЭТО

**Arbitrum** — ведущий Layer 2-масштабировщик Ethereum на базе технологии **Optimistic Rollup**, разработанный командой **Offchain Labs** (основана в 2018). Цель — дешёвые и быстрые транзакции, совместимые с EVM, без изменения пользовательского опыта по сравнению с Ethereum.

- **Для кого:** DeFi-пользователи, разработчики dApps, институциональные игроки (RWA-сектор)
- **Запуск:** Mainnet — август 2021, токен ARB — 23 марта 2023 (TGE + airdrop)
- **Реальные пользователи:** ✅ — 2.4M MAU (avg 365d), 250+ протоколов, крупнейшие dApps: GMX, Uniswap V3, Gains Network; институциональные интеграции BlackRock (BUIDL), Franklin Templeton (BENJI), Robinhood (tokenized stocks)
- **Ecosystem:** Orbit framework (100+ дочерних chains), Arbitrum Nova (для высоконагруженных приложений), Stylus VM (WASM-совместимость)
- **ARB-токен:** governance-only, используется исключительно для голосований в DAO

## ТЕХНОЛОГИЯ

**Архитектура:**
- **Optimistic Rollup** — транзакции батчируются на L2, публикуются на Ethereum L1; валидность предполагается по умолчанию (без ZK-доказательств), fraud proof-окно — 7 дней
- **ArbOS** — кастомная операционная система (последнее обновление: ArbOS 50 Dia, октябрь 2025, с поддержкой Ethereum Fusaka hard fork)
- **Stylus** — второй VM на базе WASM, позволяет писать смарт-контракты на Rust/C/C++ в дополнение к Solidity
- **Orbit framework** — L3-экосистема, позволяет запускать собственные chains поверх Arbitrum One/Nova
- **Timeboost** (октябрь 2025) — аукцион MEV-приоритизации транзакций (аналог PBS), принёс $6M+ к октябрю 2025

**Ключевые инновации:**
- Sequencer decentralization roadmap (в процессе)
- BOLD (Bounded Liquidity Delay) — новый challenge-протокол для более безопасных fraud proofs
- Interoperability layer (кросс-chain коммуникации)

**Зрелость продукта:** ✅ Production-ready, Mainnet работает с 2021 г., миллиарды $TVL обработано

**Аудиты безопасности:** Проведены несколькими ведущими фирмами (Trail of Bits, ABDK, Certik); конкретные отчёты по ArbOS — публичны на GitHub

**Слабость технологии:** Optimistic Rollup проигрывает ZK-Rollup по финальности (7 дней vs минуты) и long-term scalability — это системный технологический риск по мере зрелости ZK-решений
